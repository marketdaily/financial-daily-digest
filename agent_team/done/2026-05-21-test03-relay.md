本次執行環境為雲端 CCR(Claude Code Remote)環境。環境特徵:容器為每次 session 全新啟動的隔離容器,repo 由遠端 clone 而來;`git push` 被 `CCR_TEST_GITPROXY` 代理封鎖,回傳 403,無法直接推送到 GitHub。

**2. 為何走 relay_push.py 而非 git push**

雲端 CCR 環境的網路出口受限,`git push` 嘗試直接向 GitHub 推送時被代理攔截並回傳 403。`relay_push.py` 改以 HTTP POST 的方式,將 `origin/main..HEAD` 在 `agent_team/` 底下的所有 commit 差異打包傳送給寫入代理伺服器,由代理代為執行 `git commit + push` 並觸發完成通知信,繞過了直接 git push 的網路限制。
